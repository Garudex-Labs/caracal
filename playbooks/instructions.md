- Scope: `playbooks/` contains end-user Caracal playbooks and contributor rules for maintaining them.
- Must keep `README` user-facing and keep `instructions.md` contributor-facing.
- Must keep the layer directories limited to `sdks` and `compliance`.
- Must keep supported agent directories limited to `claude-code`, `github-copilot`, and `openai-codex`.
- Must write playbooks as copy-paste guidance for a user's own project, not for working inside the Caracal repository.
- Must use concise Markdown with direct instructions and clear placeholders.
- Must keep Caracal concepts consistent: grants define available access, policies define runtime conditions, policy sets activate policy versions.
- Must use placeholder values for credentials, identifiers, resource names, scopes, URLs, and tenant details.
- Must not put contributor rules, repository maintenance notes, or internal implementation planning in user-facing playbooks.
- Must not add unsupported agent names, layers, install flows, or generated-tool claims.
- Must not instruct users to paste secrets, tokens, private keys, or provider credentials into an AI chat.
- Must not duplicate full content across layers when a layer-specific playbook can link the concept briefly.
