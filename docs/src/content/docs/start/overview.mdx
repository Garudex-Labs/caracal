## What Caracal is

**Caracal** gates every AI agent action behind a policy-evaluated signed authority object called a [Mandate](/concepts/mandate/). Before an agent calls any API, it must declare what it intends to do. Caracal's STS evaluates that declaration against your [Policy](/concepts/policy/) and — if permitted — issues a short-lived mandate. The [Gateway](/architecture/services/) enforces the mandate on every outbound request.

No mandate, no request. No exceptions.

## The problem it solves

AI agents running with ambient credentials are a liability:

- A compromised credential is a permanent, unconstrained breach.
- There is no record of *what* the credential was used for, only *that* it was used.
- You cannot revoke a specific action mid-flight without revoking the entire credential.
- Sub-agents receiving credentials from an orchestrator inherit unrestricted authority.

Caracal replaces ambient credentials with scoped, time-bounded mandates. Every action is declared before it runs, policy-evaluated, signed, and recorded in an [append-only Ledger](/concepts/ledger/).

## Core primitives

| Primitive | What it is |
|---|---|
| [Principal](/concepts/principal/) | The identity requesting authority: application, user, or delegated agent. |
| [Policy](/concepts/policy/) | OPA Rego rules that decide what a principal may do. |
| [Mandate](/concepts/mandate/) | Short-lived ES256-signed JWT binding principal, policy decision, and intent. |
| [Intent](/concepts/intent/) | The declared action: verb, resource, context. |
| [Caveat](/concepts/caveat/) | Constraint narrowing a mandate's scope or TTL. |
| [Delegation](/concepts/delegation/) | Scoped authority transfer from one principal to another. |
| [Workspace](/concepts/workspace/) | The policy and principal boundary. |
| [Zone](/concepts/zone/) | Deployment unit owning signing keys and the provider credential vault. |
| [Ledger](/concepts/ledger/) | Append-only record of every authority decision. |

## When to use Caracal

Use Caracal when you need:

- **AI agents that must not act without explicit policy authorization.** Every action is policy-gated before it runs.
- **Multi-agent systems with controlled authority delegation.** Sub-agents receive only the authority they need for their task.
- **Provider credential vaulting.** Agents call OpenAI, Anthropic, or any HTTP API without holding the API key.
- **Tamper-evident audit trails.** Every authority decision is recorded and replayable.
- **Zero-standing-privilege for automation.** Credentials expire; there are no long-lived tokens.

## What Caracal is not

- Caracal does not replace your identity provider (Okta, Auth0, etc.). It consumes identities from those systems as principals.
- Caracal does not enforce in-process actions (file system, memory). It enforces at the network boundary, for registered resources routed through the gateway.
- Caracal does not make policy decisions for you. You write the OPA Rego rules; Caracal evaluates them.

## Architecture in one line

Three services: **STS** (token issuer, port 8082) → **Gateway** (reverse proxy, port 8081) ← **Coordinator** (session manager, port 8080). All written in Go. SDKs for TypeScript, Python, and Go.
