## Prerequisites

- Caracal CLI installed ([Installation](/start/installation/))
- Docker Engine 24+ running
- `curl` for the HTTP call in step 5

## Overview

**Before:** no running Caracal stack, no policy, no mandates.

**After:** a running stack with a workspace, an OPA policy, a registered application, and one mandate issued and verified.

**Audience:** developers evaluating Caracal, operators setting up their first deployment.

// CARACAL_COORDINATOR_URL=http://localhost:8080
// CARACAL_GATEWAY_URL=http://localhost:8081
// CARACAL_APP_ID=quickstart-app
// CARACAL_APP_SECRET=dev-secret-do-not-use-in-prod
const client = new CaracalClient({
  coordinatorUrl: process.env.CARACAL_COORDINATOR_URL,
  gatewayUrl: process.env.CARACAL_GATEWAY_URL,
})

const session = await client.run({

## Outcomes

After completing this quickstart:

- The `quickstart-workspace` workspace exists with an active policy bundle.
- The `quickstart-app` application is registered as a principal.
- One `mandate.issued` event and one `mandate.verified` event appear in the ledger.
- The gateway is enforcing authority for all registered resources.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `caracal status` shows a service unhealthy | Docker not running, or port conflict | Check `docker ps`; ensure ports 8080–8083 are free. |
| `mandate.denied` in audit log | Policy does not permit the intent | Check `input.intent.action` in your Rego `allow` rule. |
| `401 mandate_expired` from gateway | Mandate TTL elapsed before the call | Call `client.run()` again; check system clock sync. |
| `403 intent_mismatch` from gateway | Request path does not match `intent.resource` | Ensure the `resource` pattern covers the actual URL path. |
