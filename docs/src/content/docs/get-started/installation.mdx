Install the released tools when you want to run Caracal as a product. Install SDK packages only in the applications that need to request or propagate Caracal authority.

## Install the released tools

The installer downloads release archives from GitHub, verifies each archive against `SHA256SUMS`, and installs:

- `caracal`: the local runtime CLI for `up`, `down`, `status`, `purge`, `run`, and `console`.
- `caracal-console`: the interactive Console for zones, applications, resources, providers, policies, sessions, agents, audit, diagnostics, and Control workflows.

### Linux and macOS

```sh
curl -fsSL https://raw.githubusercontent.com/Garudex-Labs/caracal/main/install-console.sh | sh
```

Optional hardening and pinning:

```sh
curl -fsSL https://raw.githubusercontent.com/Garudex-Labs/caracal/main/install-console.sh | \
  sh -s -- --version vYYYY.MM.DD --require-provenance
```

By default the installer writes to `~/.local/bin`. Override with `--install-dir /path/to/bin` or `CARACAL_INSTALL_DIR=/path/to/bin`.

### Windows

```powershell
$installer = "$env:TEMP\install-console.ps1"
iwr -UseBasicParsing https://raw.githubusercontent.com/Garudex-Labs/caracal/main/install-console.ps1 -OutFile $installer
powershell -ExecutionPolicy Bypass -File $installer
```

Optional hardening and pinning:

```powershell
powershell -ExecutionPolicy Bypass -File $installer -Version vYYYY.MM.DD -RequireProvenance
```

The Windows installer writes to `%LOCALAPPDATA%\Programs\caracal` by default and adds that directory to the user `Path`. Open a new shell after installation.

### Supported release platforms

| Platform | Architectures | Notes |
| --- | --- | --- |
| Linux | `amd64`, `arm64` | Requires `curl` or `wget`, `tar`, and `sha256sum` or `shasum`. |
| macOS | `amd64`, `arm64` | If Gatekeeper quarantines the binary, remove quarantine from the installed file. |
| Windows | `amd64` | Use the PowerShell installer. |

## Verify the tools

```sh
caracal --version
caracal-console --version
caracal status --help
```

Use `caracal` for local lifecycle and workload execution only. Use `caracal console` or `caracal-console` for product management.

## Local stack prerequisite

Running the OSS stack locally requires Docker 24+ with the Compose v2 plugin:

```sh
docker compose version
```

The local stack includes PostgreSQL and Redis containers; you do not need to install those databases separately for the quickstart.

## Install SDK packages

Install the SDK for the language your application uses.

| Language | Install command | Runtime requirement |
| --- | --- | --- |
| TypeScript | `npm install @caracalai/sdk` or `pnpm add @caracalai/sdk` | Node.js 24+ |
| Python | `pip install caracalai-sdk` | Python 3.14+ |
| Go | `go get github.com/garudex-labs/caracal/packages/sdk/go` | Go 1.26+ |

Additional packages are available for connector-specific work:

| Use case | Package |
| --- | --- |
| MCP transport | `@caracalai/transport-mcp` |
| Express connector | `@caracalai/mcp-express` |
| FastMCP connector | `caracalai-mcp-fastmcp` |
| Go `net/http` connector | `github.com/garudex-labs/caracal/packages/connectors/nethttp/go` |
| Admin API automation | `@caracalai/admin` |

## Build from source

Use the source tree only when contributing to Caracal:

```sh
git clone https://github.com/Garudex-Labs/caracal.git
cd caracal
corepack enable
corepack prepare pnpm@11.1.1 --activate
pnpm install --frozen-lockfile
```

Source-tree runtime commands are covered in [Contributor Quickstart](./contributor-quickstart/).
