This page shows the smallest SDK integration in TypeScript, Python, and Go. Each example calls a Gateway-protected resource as the configured application. The SDK provisions the bounded Sessions and Delegation required for the call and leaves authorization and action-result audit behind.

Complete [First Protected Call](/get-started/first-protected-call/) first so you have runtime configuration, a resource ID, and a protected path.

## Author the Runtime Profile

First Protected Call leaves you with a zone ID, an application ID, a client secret file, and a resource. All SDKs read those values from a `caracal.toml` profile or from `CARACAL_*` environment variables. The profile is a file you author with the IDs from guided setup - guided setup does not write it for you:

```toml
zone_id = "zone_prod"
application_id = "app_agent"
app_client_secret_file = "/path/to/caracal-client-secret"

[[credentials]]
resource = "resource://example-api"
upstream_prefix = "http://caracalDemoUpstream:80"
```

On the local stack the SDK defaults cover the STS, Coordinator, and Gateway URLs. The profile must name the owner-only client-secret file explicitly; the SDK never searches local credential directories. The secret comes from application creation and can be revealed from the application's detail panel, with every reveal audited. For cloud or custom deployments add `sts_url`, `coordinator_url`, and `gateway_url`; see [Configuration File](/runtime-console/config-file/) for the full schema.

The profile identifies the workload, not each agent. `zone_id` and `application_id` name the one confidential application your service runs as; every Session the SDK starts executes under that application identity, so one profile serves all the agents your workload creates.

Point the examples at the profile and choose which configured resource and path they call:

```sh
export CARACAL_CONFIG=/path/to/caracal.toml
export CARACAL_RESOURCE_ID=resource://example-api
export CARACAL_RESOURCE_PATH=/
export CARACAL_RESOURCE_SCOPE=example:read
```

On Windows PowerShell:

```powershell
$env:CARACAL_CONFIG = "C:\path\to\caracal.toml"
$env:CARACAL_RESOURCE_ID = "resource://example-api"
$env:CARACAL_RESOURCE_PATH = "/"
$env:CARACAL_RESOURCE_SCOPE = "example:read"
```

## Install the SDK

Install the package for your application language.

| Language   | Install command                                           | Runtime requirement |
| ---------- | --------------------------------------------------------- | ------------------- |
| TypeScript | `npm install @caracalai/sdk` or `pnpm add @caracalai/sdk` | Node.js 22+         |
| Python     | `pip install caracalai-sdk`                               | Python 3.12+        |
| Go         | `go get github.com/garudex-labs/caracal/packages/sdk/go`  | Go 1.26+            |

## TypeScript

```typescript
import { Caracal } from '@caracalai/sdk'

const caracal = new Caracal()
const resourceId = process.env.CARACAL_RESOURCE_ID
const resourcePath = process.env.CARACAL_RESOURCE_PATH
const resourceScope = process.env.CARACAL_RESOURCE_SCOPE

if (!resourceId) throw new Error('CARACAL_RESOURCE_ID is required')
if (!resourcePath) throw new Error('CARACAL_RESOURCE_PATH is required')
if (!resourceScope) throw new Error('CARACAL_RESOURCE_SCOPE is required')

const governedFetch = caracal.applicationTransport(resourceId, {
    scopes: [resourceScope],
})
const target = caracal.gatewayRequest(resourceId, resourcePath)
try {
    const response = await governedFetch(target.url, {
        method: 'GET',
    })
    if (!response.ok) throw new Error(`protected call failed: ${response.status}`)
    console.log(await response.text())
} finally {
    await caracal.close()
}
```

## Python

```python
import asyncio
import os

from caracalai import Caracal

caracal = Caracal()
resource_id = os.environ["CARACAL_RESOURCE_ID"]
resource_path = os.environ["CARACAL_RESOURCE_PATH"]
resource_scope = os.environ["CARACAL_RESOURCE_SCOPE"]

async def main():
    target = caracal.gateway_request(resource_id, resource_path)
    try:
        async with caracal.application_transport(
            resource_id,
            scopes=[resource_scope],
        ) as governed:
            response = await governed.get(target.url)
            response.raise_for_status()
            print(response.text)
    finally:
        await caracal.aclose()

asyncio.run(main())
```

## Go

```go
package main

import (
    "fmt"
    "io"
    "os"

    caracal "github.com/garudex-labs/caracal/packages/sdk/go"
)

func main() {
    c, err := caracal.New()
    if err != nil {
        panic(err)
    }

    resourceID := os.Getenv("CARACAL_RESOURCE_ID")
    resourcePath := os.Getenv("CARACAL_RESOURCE_PATH")
    resourceScope := os.Getenv("CARACAL_RESOURCE_SCOPE")
    if resourceID == "" || resourcePath == "" || resourceScope == "" {
        panic("CARACAL_RESOURCE_ID, CARACAL_RESOURCE_PATH, and CARACAL_RESOURCE_SCOPE are required")
    }
    defer c.Close()

    governed, err := c.ApplicationTransport(nil, resourceID, caracal.ApplicationTransportOptions{
        Scopes: []string{resourceScope},
    })
    if err != nil {
        panic(err)
    }
    target, err := c.GatewayRequest(resourceID, resourcePath)
    if err != nil {
        panic(err)
    }
    resp, err := governed.Get(target.URL)
    if err != nil {
        panic(err)
    }
    defer resp.Body.Close()
    if resp.StatusCode < 200 || resp.StatusCode >= 300 {
        panic(fmt.Sprintf("protected call failed: %s", resp.Status))
    }
    body, _ := io.ReadAll(resp.Body)
    fmt.Println(string(body))
}
```

## SDK Calls

- `new Caracal()` / `Caracal()` / `New()` loads runtime configuration and prepares token exchange.
- `applicationTransport()` / `application_transport()` / `ApplicationTransport()` builds application-owned, Delegation-bounded authority and returns a transport pinned to one resource.
- `gatewayRequest()` / `GatewayRequest()` builds the Gateway URL and `X-Caracal-Resource` routing header.
- `transport()` / `Transport()` mints scoped Gateway authority, injects context, and applies resource routing.

`applicationTransport()` is the smallest application-owned call path. It provisions a source and target Session, narrows the requested scope over a Delegation, and mints a fresh replay-protected Gateway mandate for each request. For agent work that already runs under delegated authority, use `session()` with `fetch(..., { scopes })`; see the language SDK guide before switching paths. You reuse one configured application for all Sessions your workload starts; you do not register an application per agent. See [Should I create one application per agent?](/reference/faq/#faq-006).

If policy denies the exchange, STS returns HTTP 403 and the SDK surfaces an error. Open web console **Audit** or **explain** with the request ID to see the policy and diagnostics that determined the result.
