This page shows the smallest SDK integration in TypeScript, Python, and Go. Each example opens a Session, calls a Gateway-protected resource, and leaves authorization and action-result audit behind.

Complete [First Protected Call](/get-started/first-protected-call/) first so you have runtime configuration, a resource ID, and a protected path.

## Author the Runtime Profile

First Protected Call leaves you with a zone ID, an application ID, a client secret file, and a resource. All SDKs read those values from a `caracal.toml` profile or from `CARACAL_*` environment variables. The profile is a file you author with the IDs from guided setup - guided setup does not write it for you:

```toml
zone_id = "zone_prod"
application_id = "app_agent"

[[credentials]]
env = "EXAMPLE_API_TOKEN"
resource = "resource://example-api"
upstream_prefix = "http://caracalDemoUpstream:80"
```

On the local stack this is the whole file: the SDK defaults cover the STS, Coordinator, and Gateway URLs, and the client secret is read from the owner-only file you create at `<Caracal config dir>/runtime/<zone_id>/<application_id>/client-secret`, holding the secret from when the application was created - retrievable any time from the application's detail panel, with every reveal audited. For cloud or custom deployments add `zone_url`, `coordinator_url`, `gateway_url`, and `app_client_secret_file`; see [Configuration File](/runtime-console/config-file/) for the full schema.

The profile identifies the workload, not each agent. `zone_id` and `application_id` name the one confidential application your service runs as; every Session the SDK starts executes under that application identity, so one profile serves all the agents your workload creates.

Point the examples at the profile and choose which configured resource and path they call:

```sh
export CARACAL_CONFIG=/path/to/caracal.toml
export CARACAL_RESOURCE_ID=resource://example-api
export CARACAL_RESOURCE_PATH=/
```

On Windows PowerShell:

```powershell
$env:CARACAL_CONFIG = "C:\path\to\caracal.toml"
$env:CARACAL_RESOURCE_ID = "resource://example-api"
$env:CARACAL_RESOURCE_PATH = "/"
```

## Install the SDK

Install the package for your application language.

| Language   | Install command                                           | Runtime requirement |
| ---------- | --------------------------------------------------------- | ------------------- |
| TypeScript | `npm install @caracalai/sdk` or `pnpm add @caracalai/sdk` | Node.js 22+         |
| Python     | `pip install caracalai-sdk`                               | Python 3.12+        |
| Go         | `go get github.com/garudex-labs/caracal/packages/sdk/go`  | Go 1.26+            |

## TypeScript

```typescript
import { Caracal } from '@caracalai/sdk'

const caracal = new Caracal()
const resourceId = process.env.CARACAL_RESOURCE_ID
const resourcePath = process.env.CARACAL_RESOURCE_PATH

if (!resourceId) throw new Error('CARACAL_RESOURCE_ID is required')
if (!resourcePath) throw new Error('CARACAL_RESOURCE_PATH is required')

await caracal.session(async () => {
  const ctx = caracal.current()
  console.log('Session:', ctx?.agentSessionId)

  const response = await caracal.fetch(resourceId, resourcePath, {
    method: 'GET',
  })

  console.log(await response.text())
})
```

## Python

```python
import asyncio
import os

from caracalai import Caracal

caracal = Caracal()
resource_id = os.environ["CARACAL_RESOURCE_ID"]
resource_path = os.environ["CARACAL_RESOURCE_PATH"]

async def main():
    async with caracal.session() as ctx:
        print("Session:", ctx.session_id)

        response = await caracal.fetch(
            resource_id,
            resource_path,
            method="GET",
        )
        print(response.text)

asyncio.run(main())
```

## Go

```go
package main

import (
    "context"
    "fmt"
    "io"
    "net/http"
    "os"

    caracal "github.com/garudex-labs/caracal/packages/sdk/go"
)

func main() {
    c, err := caracal.New()
    if err != nil {
        panic(err)
    }

    resourceID := os.Getenv("CARACAL_RESOURCE_ID")
    resourcePath := os.Getenv("CARACAL_RESOURCE_PATH")
    if resourceID == "" || resourcePath == "" {
        panic("CARACAL_RESOURCE_ID and CARACAL_RESOURCE_PATH are required")
    }

    err = c.Session(context.Background(), func(ctx context.Context) error {
        cc, _ := c.Current(ctx)
        fmt.Println("Session:", cc.AgentSessionID)

        resp, err := c.Fetch(ctx, http.MethodGet, resourceID, resourcePath)
        if err != nil {
            return err
        }
        defer resp.Body.Close()

        body, _ := io.ReadAll(resp.Body)
        fmt.Println(string(body))
        return nil
    })
    if err != nil {
        panic(err)
    }
}
```

## SDK Calls

| Call                                    | Role                                                                                                            |
| --------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| `new Caracal()` / `Caracal()` / `New()` | Loads the runtime profile or environment configuration and prepares token exchange.                             |
| `session()` / `Session()`               | Creates a Session, binds context for the work, and terminates the session when the block exits.                 |
| `fetch()` / `Fetch()`                   | Sends one request to a resource through the Gateway with context and authority injected.                        |
| `gatewayRequest()` / `GatewayRequest()` | Builds the Gateway URL and `X-Caracal-Resource` header for explicit resource routing.                           |
| `transport()` / `Transport()`           | Injects Caracal trace and baggage context on outbound calls and attaches authorization on Gateway-routed calls. |

`session()` creates a Session under the application from your runtime profile. You reuse that one application for every Session your workload starts; you do not register an application per agent. See [Should I create one application per agent?](/reference/faq/#faq-006).

If policy denies the exchange, STS returns HTTP 403 and the SDK surfaces an error. Open web console **Audit** or **explain** with the request ID to see the policy and diagnostics that determined the result.
