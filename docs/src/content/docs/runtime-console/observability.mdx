Observability workflows live in the Console because they need operator context, selected zones, credentials, and structured API responses.

## Diagnostics

Open `diagnostics` with `d`. The Console Doctor view runs the shared engine diagnostics across four sections:

| Section | Checks |
| --- | --- |
| `health` | API health and basic management reachability. |
| `readiness` | Service readiness, dependencies, metrics, and runtime endpoints. |
| `zones` | Visible zones, resources, policy sets, grants, and audit queryability. |
| `preflight` | Local deployment and operator environment checks. |

Useful Doctor controls include reload, all-zones or selected-zone checks, preflight-only checks, and strict readiness where warnings fail automation gates.

## Audit

Open `audit` with `a`. Audit records tie STS, Gateway, Coordinator, policy, and Control decisions to request IDs and authority anchors. Use the audit view to filter recent events, inspect denials, and gather incident context.

## Explain

Open `explain` with `e` when you have a request ID and need the decision trace. Explanation output connects the request, selected resource, subject, scopes, determining policies, diagnostics, and result.

## Credential inspection

Open `credential` with `c` to read a protected-resource token or inspect a JWT locally. Local inspection helps compare issuer, subject, zone, session, target resources, scopes, expiry, algorithm, key ID, and token ID against the audit trail. Resource servers must still verify signatures and revocation; local decoding is not authorization.

## Troubleshooting

| Symptom | Check |
| --- | --- |
| Diagnostics cannot reach API | Run `caracal status --ready`; custom deployments should confirm the Management API URL override. |
| Zone diagnostics show no zones | Confirm the admin token has visibility into the expected zone. |
| Audit records are missing | Confirm the selected zone, time window, and audit service readiness. |
| Credential inspection succeeds but service rejects the token | Check audience/resource, scopes, token use, revocation state, and identity verifier settings. |
