Workload configuration has two consumers. `caracal run` needs only a workload identity; its credential bindings live in the web console. SDK credential loaders read a local profile. Neither is Console state and neither creates, selects, or manages zones.

## Workload Identity for caracal run

`caracal run` authenticates as one confidential application and fetches that application's launch bindings from STS. The environment carries only identity:

| Variable | Meaning |
| --- | --- |
| `CARACAL_APPLICATION_ID` | Confidential application identifier. Required. |
| `CARACAL_APP_CLIENT_SECRET` | Inline local-development client secret. |
| `CARACAL_APP_CLIENT_SECRET_FILE` | Cloud/custom path to a mounted client-secret file. |
| `CARACAL_STS_URL` | Cloud/custom STS URL override for token exchange. |

Set only one of `CARACAL_APP_CLIENT_SECRET` and `CARACAL_APP_CLIENT_SECRET_FILE`. Local dev and stable service URLs resolve automatically; set `CARACAL_STS_URL` only for cloud deployments, containers, or custom service locations.

When neither secret variable is set, local dev and stable launches read the owner-only secret file at:

```text
<Caracal config dir>/runtime/<application_id>/client-secret
```

The OS Caracal config directory is `$XDG_CONFIG_HOME/caracal` (or `~/.config/caracal`) on Linux, `~/Library/Application Support/Caracal` on macOS, and `%APPDATA%\Caracal` on Windows. Set `CARACAL_CONFIG_HOME` to override it on any platform.

You create the secret file yourself. The web console shows the one-time client secret at application creation but never writes to your machine:

```bash
mkdir -p ~/.config/caracal/runtime/<application_id>
printf '%s' '<client secret>' > ~/.config/caracal/runtime/<application_id>/client-secret
chmod 600 ~/.config/caracal/runtime/<application_id>/client-secret
```

The default path is read only in local dev and stable modes; production launches (`NODE_ENV=production`) require explicit secret configuration.

## Launch Bindings

Which environment variables `caracal run` injects, which resources fill them, the credential type, the TTL, and optional-credential behavior are all part of the application's run manifest, authored on the **Run** page in the web console. STS serves the manifest to the workload after it proves possession of the client secret, so the zone, resource identifiers, and variable names never live on the machine that launches the workload.

Every manifest is scoped to its application's zone, and each save is attributed: the Run page shows which console operator configured the bindings and when, and the change appears in the zone's admin audit feed under that operator's profile.

The Run page also renders the exact launch command for the application and states whether the binding profile requires the Gateway: `caracal_mandate` bindings need workload traffic pointed at the Gateway, while an all-`provider_token` profile calls providers directly. See [Run Workloads](/runtime-console/runtime/) for the launch flow.

## SDK Profile

SDK credential loaders resolve a local profile instead of calling STS for a manifest:

| Order | Source |
| --- | --- |
| 1 | `CARACAL_CONFIG`, when set and the file exists. |
| 2 | Environment runtime config (`CARACAL_ZONE_ID`, `CARACAL_APPLICATION_ID`, secret and credential variables). |
| 3 | `caracal.toml` in the OS Caracal config directory. |

SDKs do not read `./caracal.toml` from the current working directory. Local dev and stable SDK loads also read the client secret and credential manifest from `<Caracal config dir>/runtime/<zone_id>/<application_id>/client-secret` and `credentials.json`; set `CARACAL_RUN_CREDENTIALS_FILE` or `CARACAL_RUN_CREDENTIALS` (one, not both) to override the manifest location.

### TOML Schema

```toml
zone_id = "zone_prod"
application_id = "app_agent"
ttl_seconds = 900

[[credentials]]
env = "OPENAI_API_KEY"
resource = "resource://openai"
credential_type = "provider_token"
upstream_prefix = "https://api.openai.com/v1"

[[optional_credentials]]
env = "CARACAL_MANDATE"
resource = "resource://mandate-aware-api"
credential_type = "caracal_mandate"
on_failure = "warn"
```

### Field Reference

| Field | Meaning |
| --- | --- |
| `zone_url` | Cloud/custom STS URL override; `sts_url` is accepted as an SDK-readable alias/fallback. |
| `gateway_url` | Cloud/custom Gateway endpoint override for SDK transports. |
| `coordinator_url` | Cloud/custom Coordinator endpoint override for SDK agent/delegation features. |
| `zone_id` | Zone created in the web console. |
| `application_id` | Confidential application used by the workload. |
| `app_client_secret_file` | Cloud/custom secret-file path override. |
| `app_client_secret` | Inline local-development secret. |
| `ttl_seconds` | Credential exchange TTL; defaults to and is capped at 900 seconds. |
| `continue_on_failure` | Required credential failure behavior; defaults to `false`. |
| `credentials[]` | Required resource tokens. |
| `optional_credentials[]` | Optional resource tokens with `on_failure = "warn"` or `"error"`. |

Credential entries use `env`, `resource`, optional `upstream_prefix`, and optional `credential_type`. `provider_token` requires a provider with `allow_runtime_injection = true`; `caracal_mandate` is for workloads that consume Caracal mandates directly. SDKs can also use entries as resource audiences and Gateway routing hints.

## Security Rules

- Store the client secret at the runtime path with owner-only permissions for dev and stable launches.
- Use explicit secret-file paths for cloud deployments, containers, or external secret stores.
- Secret files and explicit config files must not be group- or world-writable.
- Non-local insecure `http://` STS URLs are rejected unless explicitly allowed by environment.
- Environment names that affect runtime loading, such as `NODE_OPTIONS`, `LD_PRELOAD`, and dynamic-library path variables, are blocked as binding targets.

## Troubleshooting

| Symptom | Check |
| --- | --- |
| `workload identity is required to run a command` | Set `CARACAL_APPLICATION_ID` and a client secret source. |
| `run manifest not configured` | Define launch bindings on the Run page in the web console. |
| Provider key is not injected | Confirm the binding uses the **Provider token** credential type and the provider config allows runtime injection. |
| Permission error on the secret file | The file must be owner-only; `chmod 600` it. |
| `CARACAL_CONFIG` is set but ignored (SDKs) | The file must exist; when it is set, no other profile path is considered. |
| Optional credential stops the process | Set the binding's unavailable behavior to warn, or `on_failure = "warn"` in an SDK profile. |
