Session and delegation operations are web console workflows backed by the Coordinator. They are not top-level runtime CLI commands.

## Requirements

| Input                                                           | Purpose                                                                     |
| --------------------------------------------------------------- | --------------------------------------------------------------------------- |
| Coordinator endpoint                                            | Automatic for local dev/stable; explicit only for cloud/custom deployments. |
| `CARACAL_COORDINATOR_TOKEN` or `CARACAL_COORDINATOR_TOKEN_FILE` | Coordinator authority for session and delegation views.                     |
| Selected zone                                                   | Scope for session and delegation queries.                                   |

For the default local stack, the web console prefers the Coordinator token generated by `caracal up` so a purge/up cycle does not leave an old exported token in use.

## Sessions

Open **Sessions** to list governed executions, inspect a Session tree, suspend an active subtree, resume a suspended subtree, or terminate a Session. Tree views show parent-child relationships so operators can understand inherited authority.

## Delegation

Open **Delegation** to inspect active, inbound, and outbound delegations. Detail views show source and target sessions, resources, scope bounds, expiry, and traversal state.

Revoking a delegation ends the delegated authority immediately. Review outbound delegations before terminating a session so dependent work can be stopped cleanly.

## Audit Correlation

Session IDs and Delegation IDs appear in audit events. Use **Audit** request tracing when you need the policy decision and diagnostic payload behind an allow or deny.

## Troubleshooting

| Symptom                               | Check                                                                                      |
| ------------------------------------- | ------------------------------------------------------------------------------------------ |
| Session pages are empty               | Confirm the selected zone and that workloads are creating sessions through an SDK.         |
| Coordinator token missing             | Run `caracal up` locally or set `CARACAL_COORDINATOR_TOKEN_FILE` for a remote Coordinator. |
| Delegation revocation appears delayed | Confirm resource servers consume revocation state and reject revoked delegation anchors.   |
