The Go SDK provides agent lifecycle, delegation, Gateway routing, and context propagation for Go services.

## Install

```bash
go get github.com/garudex-labs/caracal/packages/sdk/go
```

## Connect and Configure

```go
import caracal "github.com/garudex-labs/caracal/packages/sdk/go"

client, err := caracal.New()
if err != nil {
	return err
}
```

`New()` loads `CARACAL_CONFIG`, a `caracal.toml` in the default config path, or environment variables. `FromEnv()`, `FromConfig()`, and `FromClientSecret()` are available when you need explicit control.

## Core methods

| API | Purpose |
| --- | --- |
| `New()` | Auto-detect profile or environment configuration. |
| `FromEnv()` | Load from environment variables. |
| `FromConfig(path)` | Load a runtime profile. |
| `FromClientSecret(options)` | Use STS client-secret exchange for root SDK operations. |
| `client.Spawn(ctx, fn, opts...)` | Create an agent session and run `fn` with a bound `context.Context`; set `SpawnOptions.Grant` to bound the child's authority. |
| `client.Delegate(ctx, opts)` | Create a delegation edge from the current agent session; returns the edge for the receiver to adopt. |
| `client.AdoptDelegation(ctx, edgeID)` | Present a received delegation edge: derive a context carrying it. |
| `client.Headers(ctx, opts...)` | Project the bound context to `http.Header`. |
| `client.BindFromRequest(ctx, req, opts...)` | Bind inbound Caracal envelope headers. |
| `client.Transport(base, opts...)` | Return an `*http.Client` that injects Caracal headers. Set `RootOptions.Scopes` to authorize gateway calls with a scoped resource mandate instead of the raw subject token. |
| `client.MintMandate(ctx, resourceID, scopes, opts...)` | Mint a cached resource mandate carrying the bound agent session and delegation edge; requires client-secret credentials. |
| `client.WaitForApproval(ctx, challengeID, timeout)` | Long-poll an approval challenge raised by an approval-gated mint; returns the final state. |
| `client.GatewayRequest(resourceID, path)` | Build explicit Gateway routing metadata. |
| `client.Current(ctx)` | Inspect the current Caracal context. |

## Context Propagation

```go
constraints := &caracal.DelegationConstraints{
	MaxHops:        1,
	Budget:         5,
	PolicyApproved: true,
}
```

Available fields are `Resources`, `MaxDepth`, `MaxHops`, `TTLSeconds`, `Budget`, `PolicyApproved`, `ExpiresAt`, and `BroadReason`.

## Protect Inbound Requests

Use `BindFromRequest()` to propagate a Caracal context after an upstream Gateway, adapter, or verify-engine verifier has accepted the inbound mandate. Set `RootOptions.Verify` to enforce the bearer token at the boundary itself; the callback receives the inbound token and must return an error on failure. Return a `*VerifiedClaims` value from the callback to stamp token-proven attribution (zone, application, agent session, delegation edge, hop) over the caller-supplied envelope headers. Use [Verification Layer Overview](/sdks/verification-layer/) when the Go service must enforce mandate signatures, scopes, targets, agent identity, delegation, or revocation at its own boundary.

`Headers` and `Transport` require a bound Caracal context by default. Pass `caracal.RootOptions{AllowRoot: true}` only when the call should intentionally use the application subject token.

## Errors and Observability

STS denials return `*oauth.CaracalError` carrying `Code`, `HTTPStatus`, and `RequestID`, so callers branch with `errors.As` instead of matching message text. Approval-gated exchanges return `*oauth.InteractionRequiredError` with the challenge fields, and coordinator failures return `*caracal.CoordinatorError` with `StatusCode`, `Method`, and `Path`.

```go
import oauth "github.com/garudex-labs/caracal/packages/oauth/go"

_, err := client.MintMandate(ctx, "https://api.example.com/tickets", []string{"tickets:read"})
var denied *oauth.CaracalError
if errors.As(err, &denied) && denied.Code == "access_denied" {
	log.Printf("denied by policy (request %s)", denied.RequestID)
}

client.OnEvent(func(event oauth.Event) {
	metrics.Timing("caracal."+event.Type, event.Duration, event.Ok)
})
```

`OnEvent(hook)` reports every control-plane operation: `token.exchange` (with `Resources`, `Scopes`, and `Cached` for cache hits), `approval.wait` (with `ChallengeID` and the final `State`), and `coordinator.call` (with `Method`, `Path`, and `Status`). Each event carries `Ok` and `Duration`, ready to bridge into any metrics or tracing system. A hook that panics is recovered and never disturbs the operation that emitted the event.
