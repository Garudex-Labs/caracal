`@caracalai/sdk` is the main TypeScript package for agent lifecycle, delegation, Gateway routing, and Caracal context propagation.

## Install

```bash
npm install @caracalai/sdk
```

The package is ESM-only and targets Node `>=22`. To consume it from a CommonJS project, use a dynamic `await import('@caracalai/sdk')` or set `"type": "module"` in your `package.json`.

## Connect and Configure

| API | Use it when |
| --- | --- |
| `new Caracal()` | You want auto-detection from explicit options, `CARACAL_CONFIG`, the default profile, or environment variables. |
| `Caracal.fromEnv()` | You want environment-only startup. |
| `Caracal.fromConfig(path)` | You want to load a runtime profile. |
| `Caracal.fromClientSecret(options)` | You want service-root token refresh through STS client-secret exchange, from a static credential triple or a `credentials` resolver. |

```ts
import { Caracal } from "@caracalai/sdk";

const caracal = new Caracal();
```

`Caracal.fromClientSecret` accepts either a static `zoneId`/`applicationId`/`clientSecret` triple or a `credentials` resolver: a function returning `{ zoneId, applicationId, clientSecret }`, resolved on every exchange. Use the resolver when the credential is provisioned after startup or rotates while the process runs - the next exchange simply presents whatever the resolver returns, and a resolver that returns `null` fails the call closed with `CredentialsUnavailableError` instead of using a stale secret. When the resolved identity changes, the client rebuilds its exchange state so no cached token or mandate crosses identities. `resources` is optional for a client that only runs governed transports; `spawn` and lifecycle calls require at least one.

## Core methods

| Method | Purpose |
| --- | --- |
| `spawn(fn, options?)` | Create an agent session and bind context while `fn` runs; pass `grant: Grant.narrow(...)` to bound the child's authority. |
| `spawnService(options?)` | Start a long-lived service agent; auto-renews its lease and returns a handle with `heartbeat()` and `close()`. Pass `grant: Grant.narrow(...)` to bound the handle's authority. |
| `delegate(options)` | Create a delegation edge to an existing agent session; returns the edge for the receiver to adopt. |
| `adoptDelegation(edgeId, fn)` | Present a received delegation edge: bind a derived context carrying it while `fn` runs. |
| `headers(options?)` | Project the current bound context into HTTP headers synchronously. |
| `headersAsync(options?)` | Project headers when a root token may require async refresh. |
| `bindFromHeaders(headers, fn, options?)` | Bind inbound Caracal envelope headers to the current async context. |
| `transport(options?)` | Return a fetch-compatible function that injects Caracal headers and Gateway routing. Pass `scopes` to authorize gateway calls with a scoped resource mandate instead of the raw subject token. |
| `governedTransport(resourceId, options)` | Return a fetch pinned to one governed resource on the application's own authority: each mint cycle spawns a session pair, narrows a delegation to `scopes`, and caches the minted mandate until expiry; requires client-secret credentials. Session `labels` default to the application id, the same role `ensureGrants` authors, so a grant and its transport align without naming a role on either side. |
| `mintMandate(resourceId, scopes, options?)` | Mint a cached resource mandate carrying the bound agent session and delegation edge; requires client-secret credentials. |
| `waitForApproval(challengeId, options?)` | Long-poll an approval challenge raised by an approval-gated mint; returns the final state. |
| `gatewayRequest(resourceId, path?)` | Build a Gateway URL and `X-Caracal-Resource` header. |
| `current()` | Return the currently bound `CaracalContext`, if present. |

## Context Propagation

```ts
import { Grant } from "@caracalai/sdk";

await caracal.spawn(async () => {
  await fetch("https://api.example.com/tickets", {
    headers: await caracal.headersAsync(),
  });
}, {
  grant: Grant.narrow(["tickets:read"], {
    resourceId: "https://api.example.com/tickets",
    constraints: {
      maxHops: 1,
      budget: 5,
      policyApproved: true,
    },
    ttlSeconds: 600,
  }),
});
```

`DelegationConstraints` uses camelCase fields: `resources`, `maxDepth`, `maxHops`, `ttlSeconds`, `budget`, `policyApproved`, `expiresAt`, and `broadReason`.

## Protect Inbound Requests

Use `bindFromHeaders()` to propagate a Caracal context after an upstream Gateway, connector, or transport verifier has accepted the inbound mandate. Pass `{ verify }` to enforce the bearer token at the boundary itself; the callback receives the inbound token and must throw on failure. Return a `VerifiedClaims` object from the callback to stamp token-proven attribution (zone, application, agent session, delegation edge, hop) over the caller-supplied envelope headers. Use [Verification Layer Overview](/sdks/verification-layer/) when the TypeScript service must enforce mandate signatures, scopes, targets, agent identity, delegation, or revocation at its own boundary.

Header and transport helpers refuse to fall back to the application root token unless you pass `{ allowRoot: true }`. Use that option only for trusted service-root ingress or setup calls. Normal agent work should run inside `spawn()`, `delegate()`, or `bindFromHeaders()`.

## Errors and Observability

STS denials throw `CaracalError` carrying `code`, `httpStatus`, and `requestId`, so callers branch on the machine-readable code instead of matching message text. Approval-gated exchanges throw `InteractionRequiredError`, a `CaracalError` subclass that adds the challenge fields, and coordinator failures throw `CoordinatorError` with `status`, `method`, and `path`. A client built on a `credentials` resolver throws `CredentialsUnavailableError` while the resolver returns no usable credential, so a not-yet-provisioned or expired identity fails closed rather than reaching the wire.

```ts
import { Caracal, CaracalError } from "@caracalai/sdk";

try {
  await caracal.mintMandate("https://api.example.com/tickets", ["tickets:read"]);
} catch (err) {
  if (err instanceof CaracalError && err.code === "access_denied") {
    console.warn(`denied by policy (request ${err.requestId})`);
  }
}

caracal.onEvent((event) => {
  metrics.timing(`caracal.${event.type}`, event.durationMs, { ok: event.ok });
});
```

`onEvent(hook)` reports every control-plane operation: `token.exchange` (with `resources`, `scopes`, and `cached` for cache hits), `approval.wait` (with `challengeId` and the final `state`), and `coordinator.call` (with `method`, `path`, and `status`). Each event carries `ok` and `durationMs`, ready to bridge into any metrics or tracing system. A hook that throws is ignored and never disturbs the operation that emitted the event.
