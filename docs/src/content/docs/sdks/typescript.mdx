`@caracalai/sdk` is the main TypeScript package for agent lifecycle, delegation, Gateway routing, and Caracal context propagation.

## Install

```bash
npm install @caracalai/sdk
```

The package is ESM-only and targets Node `>=22`. To consume it from a CommonJS project, use a dynamic `await import('@caracalai/sdk')` or set `"type": "module"` in your `package.json`.

## Connect and Configure

| API | Use it when |
| --- | --- |
| `new Caracal()` | You want auto-detection from explicit options, `CARACAL_CONFIG`, the default profile, or environment variables. |
| `Caracal.fromEnv()` | You want environment-only startup. |
| `Caracal.fromConfig(path)` | You want to load a runtime profile. |
| `Caracal.fromClientSecret(options)` | You want service-root token refresh through STS client-secret exchange, from a static credential triple or a `credentials` resolver. |

```ts
import { Caracal } from "@caracalai/sdk";

const caracal = new Caracal();
```

`Caracal.fromClientSecret` accepts either a static `zoneId`/`applicationId`/`clientSecret` triple or a `credentials` resolver: a function returning `{ zoneId, applicationId, clientSecret }`, resolved on every exchange. Use the resolver when the credential is provisioned after startup or rotates while the process runs - the next exchange simply presents whatever the resolver returns, and a resolver that returns `null` fails the call closed with `CredentialsUnavailableError` instead of using a stale secret. When the resolved identity changes, the client rebuilds its exchange state so no cached token or mandate crosses identities. `resources` is optional for a client that only runs application transports; `session` and lifecycle calls require at least one.

## Core methods

| Method | Purpose |
| --- | --- |
| `session(fn, options?)` | Run `fn` inside a governed session - a bounded identity Caracal establishes around whatever `fn` executes; pass `authority: Authority.narrow(...)` to bound its authority. |
| `startSession(options?)` | Start a governed session that outlives a block; auto-renews its lease and returns a handle with `heartbeat()` and `close()`. Pass `authority: Authority.narrow(...)` to bound the handle's authority. |
| `delegate(options)` | Delegate a slice of the current session's authority to an existing peer session; returns the delegation for the receiver to accept. |
| `acceptDelegation(delegationId, fn)` | Present a received delegation: bind a derived context carrying it while `fn` runs. |
| `headers(options?)` | Project the current bound context into HTTP headers synchronously. |
| `headersAsync(options?)` | Project headers when a root token may require async refresh. |
| `bindFromHeaders(headers, fn, options?)` | Bind inbound Caracal envelope headers to the current async context. |
| `transport(options?)` | Return a fetch-compatible function that injects Caracal headers and Gateway routing. Pass `scopes` to authorize gateway calls with a scoped resource mandate instead of the raw subject token. |
| `applicationTransport(resourceId, options)` | Return a fetch pinned to one resource, calling as the application's own identity: each mint cycle starts a session pair, narrows a delegation to `scopes`, and caches the minted mandate until expiry; requires client-secret credentials. A cycle costs five control-plane calls and runs roughly once per mandate TTL (default 15 minutes); requests in between are served from cache. Session `labels` default to the application id, the same role `ensureGrants` authors, so a grant and its transport align without naming a role on either side. |
| `mintMandate(resourceId, scopes, options?)` | Mint a cached resource mandate carrying the bound session and delegation; requires client-secret credentials. |
| `waitForApproval(approvalId, options?)` | Long-poll an approval raised by an approval-gated mint; returns the final `ApprovalState` (`approved`, `rejected`, `expired`, `consumed`, or `pending`). |
| `withApproval(fn, options?)` | Run an approval-gated operation end to end: on `ApprovalRequiredError` the client waits for the decision and, once approved, invokes `fn` again with the approval id. |
| `gatewayRequest(resourceId, path?)` | Build a Gateway URL and `X-Caracal-Resource` header. |
| `close()` | Drop cached application mandates and invalidate the exchanger's cached lifecycle token; sessions backing a released mandate retire on their own TTL. |
| `current()` | Return the currently bound `CaracalContext`, if present. |

## Context Propagation

```ts
import { Authority } from "@caracalai/sdk";

await caracal.session(async () => {
  await fetch("https://api.example.com/tickets", {
    headers: await caracal.headersAsync(),
  });
}, {
  authority: Authority.narrow(["tickets:read"], {
    resourceId: "https://api.example.com/tickets",
    constraints: {
      maxHops: 1,
      budget: 5,
      policyApproved: true,
    },
    ttlSeconds: 600,
  }),
});
```

`DelegationConstraints` uses camelCase fields: `resources`, `maxDepth`, `maxHops`, `ttlSeconds`, `budget`, `policyApproved`, `expiresAt`, and `broadReason`.

Every request sent through `transport()` carries the context envelope: W3C `traceparent`/`tracestate` plus `caracal.*` entries in the `baggage` header (session, delegation, and subject-session correlation ids). This applies to non-gateway hosts too - the envelope is how downstream Caracal-aware services rebind context - so treat those ids as visible to any host the transport touches. They are correlation identifiers, never credentials: the bearer token itself is attached only to gateway-routed calls, and the Gateway strips `caracal.*` baggage before forwarding to the upstream.

## Protect Inbound Requests

Use `bindFromHeaders()` to propagate a Caracal context after an upstream Gateway, adapter, or verify-engine verifier has accepted the inbound mandate. Pass `{ verify }` to enforce the bearer token at the boundary itself; the callback receives the inbound token and must throw on failure. Return a `VerifiedClaims` object from the callback to stamp token-proven attribution (zone, application, session, delegation, hop) over the caller-supplied envelope headers. Use [Verification Layer Overview](/sdks/verification-layer/) when the TypeScript service must enforce mandate signatures, scopes, targets, agent identity, delegation, or revocation at its own boundary.

Header and transport helpers refuse to fall back to the application root token unless you pass `{ asApplication: true }`. Use that option only for trusted service-root ingress or setup calls. Normal agent work should run inside `session()`, `delegate()`, or `bindFromHeaders()`.

## Errors and Observability

STS denials throw `CaracalError` carrying `code`, `httpStatus`, and `requestId`, so callers branch on the machine-readable code instead of matching message text. Approval-gated exchanges throw `ApprovalRequiredError`, a `CaracalError` subclass that adds the challenge fields, and coordinator failures throw `CoordinatorError` with `status`, `method`, and `path`. A client built on a `credentials` resolver throws `CredentialsUnavailableError` while the resolver returns no usable credential, so a not-yet-provisioned or expired identity fails closed rather than reaching the wire.

```ts
import { Caracal, CaracalError } from "@caracalai/sdk";

try {
  await caracal.mintMandate("https://api.example.com/tickets", ["tickets:read"]);
} catch (err) {
  if (err instanceof CaracalError && err.code === "access_denied") {
    console.warn(`denied by policy (request ${err.requestId})`);
  }
}

caracal.onEvent((event) => {
  metrics.timing(`caracal.${event.type}`, event.durationMs, { ok: event.ok });
});
```

`onEvent(hook)` reports every control-plane operation: `token.exchange` (with `resources`, `scopes`, and `cached` for cache hits), `approval.wait` (with `approvalId` and the final `state`), and `coordinator.call` (with `method`, `path`, and `status`). Each event carries `ok` and `durationMs`, ready to bridge into any metrics or tracing system. A hook that throws is ignored and never disturbs the operation that emitted the event.

Bridging into Prometheus takes one hook; the same shape feeds an OpenTelemetry meter:

```ts
import { Counter, Histogram } from "prom-client";

const operations = new Counter({
  name: "caracal_operations_total",
  help: "Caracal control-plane operations",
  labelNames: ["type", "ok"],
});
const latency = new Histogram({
  name: "caracal_operation_duration_ms",
  help: "Caracal control-plane operation latency",
  labelNames: ["type"],
});

caracal.onEvent((event) => {
  operations.inc({ type: event.type, ok: String(event.ok) });
  latency.observe({ type: event.type }, event.durationMs);
});
```

Operational warnings - lease loss, cleanup failures, an unverified inbound boundary in production - go to `console.warn` by default; pass `logger` in `CaracalConfig` to route them into your logging system instead.

Approval-gated flows resolve in one call with `withApproval`, which encapsulates the catch-wait-retry dance:

```ts
const mandate = await caracal.withApproval((approvalId) =>
  caracal.mintMandate("https://api.example.com/tickets", ["tickets:admin"], { approvalId }),
);
```

When a policy denies a mint for a session that carries no delegation, the error message appends a hint naming the cause - the session holds lifecycle-only authority - and the three remediations: `Authority.narrow`, `acceptDelegation`, or `applicationTransport`.
