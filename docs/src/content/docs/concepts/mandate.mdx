## Definition

A **Mandate** is a short-lived ES256-signed JWT issued by the Caracal STS. It is the central authority object in Caracal's enforcement model.

A mandate binds three things at issuance time:
- **Principal** — the authenticated identity that requested the mandate.
- **Intent** — the declared action the principal stated it would take.
- **Policy decision** — the STS's evaluation result: permitted, with any [Caveats](/concepts/caveat/) from constrain rules attached.

The mandate is a [RFC 8693](https://datatracker.ietf.org/doc/html/rfc8693) subject token. The SDK holds it for the duration of a session and presents it to the gateway as a bearer token on each outbound request.

## Why it exists

Without a mandate, agents carry ambient long-lived API keys. A stolen key is a permanent, unconstrained breach. A mandate is scoped to one declared intent, signed by the issuing authority, and expires by design. The blast radius of a compromised mandate is bounded: it expires within minutes and can be revoked in seconds via the revocation stream.

Mandates also make authority auditable. Every mandate issuance, verification, and revocation is recorded in the [Ledger](/concepts/ledger/). The full chain from intent declaration to upstream API call is traceable.

## Lifecycle

| State | Description | Transition |
|---|---|---|
| **Requested** | SDK calls `POST /oauth/2/token` with the declared intent and application assertion. | → Signed (if policy permits) or denied (if policy denies). |
| **Signed** | STS evaluates the intent against the workspace policy, attaches any caveats, signs the JWT with the zone's ES256 key. | → Active (returned to SDK). |
| **Active** | Mandate is held by the SDK. Gateway verifies it on each outbound request. | → Expired (TTL elapsed) or Revoked (session revocation event). |
| **Expired** | TTL elapsed. Gateway rejects the mandate with `401 mandate_expired`. | Terminal. |
| **Revoked** | Session revocation event published to `caracal.sessions.revoke`. Gateway's in-memory cache enforces rejection in real time. | Terminal. |

## JWT claims

| Claim | Type | Description |
|---|---|---|
| `sub` | string | Principal ID. Format: `app:<application_id>` or `delegate:<session_id>`. |
| `iss` | string | `caracal-sts/<zone_id>` |
| `jti` | string | Unique token ID. Used for replay detection (Redis SETNX). |
| `iat` | number | Issued-at timestamp (Unix seconds). |
| `exp` | number | Expiry timestamp (Unix seconds). Default TTL: 300 s. |
| `workspace_id` | string | Workspace the principal belongs to. |
| `zone_id` | string | Zone whose signing key was used. |
| `intent` | object | The declared intent: `{ action, resource, context }`. |
| `caveats` | array | Caveat constraints attached by policy constrain rules. |
| `application_id` | string | The registered application ID. |

## Examples

**Request a mandate and execute a call — TypeScript SDK**

```typescript title="Request a mandate for file:read and execute the call"
import { CaracalClient } from '@caracalai/sdk'

// CARACAL_COORDINATOR_URL, CARACAL_GATEWAY_URL, CARACAL_APP_ID, CARACAL_APP_SECRET
const client = new CaracalClient({
  coordinatorUrl: process.env.CARACAL_COORDINATOR_URL,
  gatewayUrl: process.env.CARACAL_GATEWAY_URL,
})

// run() requests a mandate for the declared intent and wraps the session.
const session = await client.run({
  intent: { action: 'file:read', resource: '/data/reports/q1.csv' },
})

// fetch() routes through the gateway; the session bearer is the mandate.
const response = await session.fetch('https://storage.example.com/data/reports/q1.csv')
const data = await response.json()

// CaracalError.MandateExpired: mandate TTL elapsed, call client.run() again
// CaracalError.PolicyDenied: policy denied the intent, review workspace policy
// CaracalError.Revoked: session was revoked upstream, do not retry
```

**Request a mandate with caveats — TypeScript SDK**

```typescript title="Mandate with a time-bounded caveat from policy constrain rules"
import { CaracalClient } from '@caracalai/sdk'

const client = new CaracalClient({
  coordinatorUrl: process.env.CARACAL_COORDINATOR_URL,
  gatewayUrl: process.env.CARACAL_GATEWAY_URL,
})

// The workspace policy has a constrain rule that attaches a 60-second TTL caveat
// for any intent targeting /data/sensitive/. The SDK transparently handles the
// shorter TTL and auto-refreshes the mandate before the caveat window closes.
const session = await client.run({
  intent: { action: 'file:read', resource: '/data/sensitive/keys.json' },
})

const response = await session.fetch('https://storage.example.com/data/sensitive/keys.json')
```

## Constraints and limits

- **No long-lived mandates.** The maximum TTL is operator-configured but cannot exceed the zone's `max_mandate_ttl` (default: 3600 seconds). The default TTL is 300 seconds.
- **One intent per mandate.** A mandate carries exactly one declared intent. If an agent needs to take two different actions, it requests two mandates.
- **Replay protection.** Each `jti` is recorded in Redis on first use. A second gateway request with the same `jti` is rejected, even within the TTL window.
- **Zone-scoped signature.** The gateway verifies the signature against the JWKS for the specific `zone_id` in the mandate. Cross-zone tokens are rejected.
- **35-second expiry buffer.** The gateway rejects mandates within 35 seconds of their `exp` claim to account for clock skew and network latency.
