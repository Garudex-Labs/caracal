---
title: Run an Agent with caracal run
description: Use a runtime profile to inject scoped Caracal access into a subprocess.
---

`caracal run` starts a local subprocess with Caracal runtime environment variables and profile-backed credentials. Use it for development, demos, and controlled local runs.

## Prepare a profile

1. Run `caracal up`.
2. Sign in to the web console and use **Guided setup** to create the zone, application, resource, and policy.
3. Copy the application's one-time client secret.
4. Author a `caracal.toml` profile with the zone, application, secret file, and a `[[credentials]]` entry for each resource. See [Configure Workloads](/runtime-console/config-file/).

The profile is the source for SDK client constructors and `caracal run`.

## Run a command

```bash
caracal run -- npm start
```

The launcher exchanges each `[[credentials]]` entry for a short-lived credential and injects the results into the configured environment variables. The child environment is otherwise scrubbed: `CARACAL_*` configuration variables stay with the launcher, and only a small allowlist such as `PATH`, `HOME`, locale, and `XDG_*` directories is inherited. SDK-based children read the profile from the default config path rather than from environment variables. Credentials are exchanged once at launch and never renewed; workloads that outlive the 15-minute TTL should use an SDK. See [Run Workloads](/runtime-console/runtime/) for the full contract.

## Validate the run

| Check | Command or surface |
| --- | --- |
| Runtime is ready | `caracal status --ready` |
| Credential is injected | `caracal run -- printenv OPENAI_API_KEY` (use your configured `env` name) |
| First request succeeds | Send a request to a configured resource path. |
| Audit captured the request | web console **Audit**. |

## Troubleshooting

| Symptom | Fix |
| --- | --- |
| SDK cannot find config | Confirm `CARACAL_CONFIG` points to your profile. |
| Secret file rejected | Restrict file permissions and avoid setting both inline and file secrets. |
| Gateway routing misses the resource | Confirm resource bindings and upstream prefixes in the profile. |
| Request denied | Check policy-set activation, scopes, and audit diagnostics. |
