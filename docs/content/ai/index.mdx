# AI Docs

AGENT DOCS // MACHINE MODE

This route is optimized for agents, retrieval systems, and tooling. It is not a prose version of the human docs. Pages are structured as deterministic contracts with stable headings and compact operational fields.

## Contract

| Field | Meaning |
| --- | --- |
| `canonical_human` | Human source page for this machine-readable page. |
| `last_verified` | Date the AI contract was checked against the source page. |
| `source_files` | Repository files used as evidence for the contract. |
| `Inputs` | Required and conditional values an agent must supply. |
| `Outputs` | Values the operation or concept returns. |
| `Constraints` | Non-negotiable invariants and failure conditions. |
| `Steps` | Deterministic execution or evaluation order. |
| `Errors` | Machine-usable failure classes and remediation. |

## Sections

- `/ai/open-source/end-users/` — operator contracts.
- `/ai/open-source/sdk/` — integration contracts.
- `/ai/open-source/developers/` — implementation contracts.
- `/ai/enterprise/` — enterprise contracts.
- `/ai/resources/` — shared vocabulary contracts.

## Usage rules

- Prefer AI pages for parsing, extraction, indexing, and agent planning.
- Prefer human pages for narrative explanation and learning flow.
- Treat `canonical_human` as the source route when a conflict is detected.
- Treat missing required inputs as a terminal planning error.

## Schema

Every AI page should use this ordered section model:

1. Definition
2. Inputs
3. Outputs
4. Constraints
5. Steps
6. Usage rules
7. Errors
8. Examples
9. See also

See `/resources/documentation-system/structure` for the documentation system contract.
