# What Caracal Open Source includes

Caracal Open Source is the self-managed Caracal runtime and host orchestrator for pre-execution authority enforcement.

## Plain-language explanation

The Open Source edition provides the host command entrypoint (`caracal`) and the runtime modules that execute authority checks before actions run.

In this repository, that split is visible as:

- `caracal-core` (`packages/caracal`) for host orchestration commands.
- `caracal-server` (`packages/caracal-server`) for runtime server modules installed in the runtime container image.

The runtime host command set includes `up`, `down`, `reset`, `purge`, `logs`, `cli`, `flow`, and `migrate`. The `up` path starts the local stack services (`postgres`, `redis`, `mcp`).

Open Source and Enterprise use different environment-variable namespaces:

- `CCL_` for Open Source core configuration.
- `CCLE_` for Enterprise-specific configuration.

## Where this appears in the system

- Package boundary: `packages/caracal/pyproject.toml` defines `caracal-core` and maps the `caracal` script to `caracal.runtime.entrypoints:caracal_entrypoint`.
- Runtime boundary: `packages/caracal-server/pyproject.toml` defines `caracal-server` as runtime server modules installed only in the runtime container image.
- Runtime implementation: `packages/caracal/caracal/runtime/` contains host entrypoints, runtime data, and hard-cut preflight checks.
- Server implementation: `packages/caracal-server/caracal/` contains CLI, core authority logic, identity, storage, flow, monitoring, provider, and enterprise modules.

## Constraints and edge cases

- Hard-cut is an operating requirement in runtime enforcement paths, not a user-facing mode switch in docs.
- In hard-cut runtime paths, SQLite is rejected and PostgreSQL is required.
- In hard-cut runtime paths, the secret backend must be explicitly managed (`vault`) and local/file-backed secret backends are rejected.
- In hard-cut runtime paths, local vault mode values (`local`, `dev`, `development`) are rejected.
- This documentation set describes self-managed operation from this repository and runtime stack commands. It does not define a hosted Caracal service.
