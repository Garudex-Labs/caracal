# policy

## Scope

Policy lifecycle management. Author, validate, and apply policy definitions.

## Operational intent

This command family is used when operators need deterministic state transitions with explicit validation and auditable outcomes.

## Typical operation sequence

1. Confirm runtime and workspace context are valid.
2. Execute command with explicit parameters and scope identifiers.
3. Validate command output and resulting state through read paths.
4. Record identifiers for downstream operations.

## Validation checklist

- Command returns expected status and structured output.
- State transitions are visible in follow-up reads.
- Errors classify input issues vs authorization vs runtime availability.

## Failure branches

- Invalid command shape or missing required identifiers.
- Authority constraints deny requested transition.
- Runtime dependency unavailable or unhealthy.
