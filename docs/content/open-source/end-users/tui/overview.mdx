# When to use Flow

When to use Flow defines part of the authority model used for authority enforcement behavior and operational usage in Caracal.

## Plain-language explanation

Caracal evaluates authority before execution. This concept describes one control point in that evaluation chain and how it interacts with policy, mandate, principal, workspace, and audit state.

## Where this appears in the system

- Runtime enforcement paths for `/open-source/end-users/tui/overview`.
- CLI and SDK operations that create, mutate, or validate authority-relevant state.
- Incident analysis and audit workflows where deterministic reconstruction is required.

## Detailed focus areas

- explicit authority preconditions.
- state transition determinism.
- observability and audit trails.
- error remediation clarity.

## Constraints and edge cases

- Missing or invalid authority context fails closed in protected operations.
- Identifier normalization must be consistent across CLI, SDK, and API boundaries.
- State transitions should be monotonic and auditable, with explicit revoke/invalidate paths where applicable.
- Cross-workspace assumptions are invalid unless explicitly delegated.
