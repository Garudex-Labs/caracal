Advanced SDK usage centers on explicit endpoint selection, custom transports, hooks, and extension registration.

**Audience:** Open-source SDK users and integrators

## Explicit Endpoint Selection

Set `CCL_API_URL` for process-wide endpoint selection:

```bash
export CCL_API_URL=http://localhost:8000
```

Or pass the endpoint directly when building a client:

```python
from caracal_sdk import CaracalClient

client = CaracalClient(api_key="sk_test_123", base_url="http://localhost:8000")
```

```ts
import { CaracalClient } from "@caracal/core";

const client = new CaracalClient({
  apiKey: "sk_test_123",
  baseUrl: "http://localhost:8000",
});
```

## Custom Transports

Both SDKs expose base, HTTP, WebSocket, and mock adapter surfaces. A custom adapter can replace API-key based HTTP initialization.

```python
client = CaracalClient(adapter=my_adapter)
```

```ts
const client = new CaracalClient({ adapter: myAdapter });
```

## Builder Pattern

Use the builder when setup needs multiple steps before client construction.

```python
from caracal_sdk import CaracalBuilder

client = (
    CaracalBuilder()
    .set_api_key("sk_test_123")
    .set_base_url("http://localhost:8000")
    .set_transport(my_adapter)
    .build()
)
```

```ts
import { CaracalBuilder } from "@caracal/core";

const client = new CaracalBuilder()
  .setApiKey("sk_test_123")
  .setBaseUrl("http://localhost:8000")
  .setTransport(myAdapter)
  .build();
```

## Extensions And Hooks

Extensions install into the SDK hook registry through `client.use(...)` or the builder's `use(...)` method. Hook behavior runs around request, response, error, and initialization events.

Keep extensions focused on client-side concerns such as instrumentation, request decoration, or local integration behavior. Identity, mandate, policy, and caveat selection remain backend-owned.
