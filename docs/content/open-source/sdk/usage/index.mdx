Use the SDK when an application needs to call registered runtime tools through Caracal's MCP service endpoint.

**Audience:** Open-source SDK users and integrators

## Client Initialization

Python:

```python
from caracal_sdk import CaracalClient

client = CaracalClient(api_key="sk_test_123")
```

Node.js / TypeScript:

```ts
import { CaracalClient } from "@caracal/core";

const client = new CaracalClient({ apiKey: "sk_test_123" });
```

Client construction requires either an API key or a custom adapter.

## Tool Calls

The primary operation is `tools.call`, which sends a request to `/mcp/tool/call`.

Python:

```python
result = await client.tools.call(
    tool_id="provider:demo:resource:jobs:action:run",
    tool_args={"job": "example"},
    correlation_id="req-123",
)
```

Node.js / TypeScript:

```ts
const result = await client.tools.call({
  toolId: "provider:demo:resource:jobs:action:run",
  toolArgs: { job: "example" },
  correlationId: "req-123",
});
```

## Scoped Calls

Use `client.context.checkout(...)` when a call should carry a workspace scope header.

Python:

```python
ctx = client.context.checkout(workspace_id="ws_1")
result = await ctx.tools.call(
    tool_id="provider:demo:resource:jobs:action:run",
    tool_args={"job": "workspace-run"},
)
```

Node.js / TypeScript:

```ts
const ctx = client.context.checkout({ workspaceId: "ws_1" });
const result = await ctx.tools.call({
  toolId: "provider:demo:resource:jobs:action:run",
  toolArgs: { job: "workspace-run" },
});
```

## Metadata Constraints

Tool-call metadata is for correlation only. Supported metadata keys are:

- `correlation_id`
- `trace_id`
- `request_id`

Caller-provided identity or policy selector fields such as `principal_id`, `mandate_id`, `policy_id`, task token claims, and caveat material are rejected by the SDK.
