Install the SDK for the language used by your integration.

**Audience:** Open-source SDK users and integrators

This page is only for application SDKs. If you need the host `caracal` command that starts the local runtime, install the runtime orchestrator package `caracal-core` or use a source checkout instead.

| Package | Language | Use it for |
| --- | --- | --- |
| `caracal-sdk` | Python | Client SDK for code that calls a running Caracal runtime. |
| `@caracal/core` | Node.js / TypeScript | Client SDK for code that calls a running Caracal runtime. |

Do not confuse the Node.js SDK package `@caracal/core` with the Python runtime package `caracal-core`. They are different packages for different ecosystems.

## Python

The Python SDK package is `caracal-sdk` and requires Python 3.10 or newer.

```bash
pip install caracal-sdk
```

The package exports `caracal_sdk`:

```python
from caracal_sdk import CaracalClient
```

## Node.js / TypeScript

The Node.js SDK package is `@caracal/core` and requires Node.js 18 or newer.

```bash
npm install @caracal/core
```

The package exports TypeScript definitions from the default package entry:

```ts
import { CaracalClient } from "@caracal/core";
```

## Runtime Endpoint

Set the runtime endpoint before creating a client:

```bash
export CCL_API_URL=http://localhost:8000
```

The default runtime compose configuration publishes the MCP service on host port `${CCL_API_PORT:-8000}` and service port `8080`. Set `CCL_API_URL` explicitly when you need deterministic endpoint selection across SDKs and environments.

## Verification

After installation, verify that the package imports and that client construction has either an API key or a custom adapter.

```python
from caracal_sdk import CaracalClient

client = CaracalClient(api_key="sk_test_123")
client.close()
```

```ts
import { CaracalClient } from "@caracal/core";

const client = new CaracalClient({ apiKey: "sk_test_123" });
client.close();
```
