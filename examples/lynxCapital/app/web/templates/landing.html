About Lynx Capital

Lynx Capital is a finance operations demo for teams that need to coordinate work across regions, systems, and approval boundaries. It is designed to show how an enterprise assistant can help with operational tasks while keeping execution visible.

Problems it addresses Manual routing, fragmented provider access, inconsistent audit trails, and slow cross-functional finance workflows.
Key capabilities Multi-agent planning, provider-backed tools, human approvals, live logs, memory, and topology visibility.
Intended users Finance operators, platform teams, risk reviewers, and product teams evaluating agent-enabled workflows.
What you will do Configure the local environment, validate provider access, then run guided finance scenarios from the demo console.

Architecture & Providers

Requests begin in the Lynx workspace, move through an orchestration layer, and use provider tools only when a task requires external or internal business data.

User request A finance operator asks for a workflow or investigation.
→
Agent plan The runtime selects regions, workflows, tools, and approval points.
→
Providers {{ overview.provider_count }} integrations cover banking, payments, ERP, tax, risk, treasury, CRM, procurement, and market data.
→
Result & audit Events stream into the console for inspection, recovery, and review.

The provider layer is intentionally diverse: REST, MCP, SDK-style, SSE, and gRPC-style surfaces are represented, but the setup flow only asks you for the configuration needed to run the local demo.

Demo Environment Notice

This environment exists to demonstrate Lynx Capital workflows and Caracal integration capabilities without touching real financial systems.

Real Application routing, agent orchestration, provider HTTP calls, event streaming, logs, setup checks, and UI state.
Simulated Provider accounts, credentials, invoices, payments, sanctions results, tax outputs, market data, and audit records.
Limitations No real money moves, no production compliance decision is made, and generated outputs require human judgment.
