Caracal setup

Run these commands once before starting the demo. Each block is a single terminal command.

0. Install Caracal▼
0
Install the Caracal CLI and SDK
pip install caracal-core caracal-sdk
This installs the caracal CLI and the Python SDK used by this demo app. Requires Python 3.10+ and Docker.
1. Start the Caracal runtime▼
1
Bring up the full stack
caracal up
Starts postgres, redis, vault, and the Caracal API on the default port (8080).
Permission denied on ~/.caracal?

If you see PermissionError: [Errno 13] Permission denied: '/home/…/.caracal/runtime', the directory was created by root (e.g. a prior docker run as root). Fix it with either option:

Option A — use a writable override (no sudo needed):

export CARACAL_HOME=~/.config/caracal

Then run caracal up again. Add the export to your shell profile to make it permanent.

Option B — reclaim ownership (requires sudo):

sudo chown -R $USER:$USER ~/.caracal
GHCR image pull denied?

If you see denied — ghcr.io/garudex-labs/caracal-runtime, authenticate with GHCR using a GitHub PAT with read:packages scope:

docker login ghcr.io -u YOUR_GITHUB_USERNAME

Then run caracal up again.

2
Run migrations
caracal migrate
2. Workspace▼
3
Create and activate workspace
caracal cli workspace create lynx-capital workspace use lynx-capital
Creates the lynx-capital workspace and sets it as the active context for all subsequent commands.
4
Set workspace and SDK auth in .env
Open examples/lynxCapital/.env and set:
CARACAL_WORKSPACE_ID=lynx-capital
CARACAL_API_KEY= — provide a valid Caracal bearer/access token for the SDK client.

14
Add provider configuration

5. Create policies▼
25
Finance Control — full provider access, delegation allowed
caracal cli policy create \ --principal-id <finance-control-uuid> \ --max-validity-seconds 28800 \ --resource-pattern "provider:*:resource:*" \ --action "provider:*:action:*" \ --allow-delegation
Replace <finance-control-uuid> with the UUID from principal list.
26
Policy Check agents — read-only compliance scope
caracal cli policy create \ --principal-id <policy-check-uuid> \ --max-validity-seconds 3600 \ --provider compliance-nexus \ --resource-pattern "provider:compliance-nexus:resource:*" \ --action "provider:compliance-nexus:action:*"
27
Payment Execution — narrow payment scope only
caracal cli policy create \ --principal-id <payment-execution-uuid> \ --max-validity-seconds 1800 \ --provider mercury-bank \ --resource-pattern "provider:mercury-bank:resource:payment" \ --action "provider:mercury-bank:action:submit_payment" policy create \ --principal-id <payment-execution-uuid> \ --max-validity-seconds 1800 \ --provider wise-payouts \ --resource-pattern "provider:wise-payouts:resource:payout" \ --action "provider:wise-payouts:action:submit_payout"
28
Regional Orchestrator — full provider access, delegation allowed
caracal cli policy create \ --principal-id <regional-orchestrator-uuid> \ --max-validity-seconds 21600 \ --resource-pattern "provider:*:resource:*" \ --action "provider:*:action:*" \ --allow-delegation
6. Issue mandates and delegate authority▼
29
Issue Finance Control root mandate
caracal cli authority mandate \ --issuer-id <finance-control-uuid> \ --subject-id <finance-control-uuid> \ --tool-id <tool-id> \ --validity-seconds 28800
Use tool list to find registered tool IDs. Note the mandate UUID from the output for the delegate step below.
30
Delegate to Regional Orchestrators (one per region)
{% for r in regions %}caracal cli authority delegate \ --source-mandate-id <finance-control-mandate-uuid> \ --target-subject-id <regional-orchestrator-uuid> \ --resource-scope "provider:*:resource:*" \ --action-scope "provider:*:action:*" \ --validity-seconds 21600 \ --context-tags "region:{{ r.id }}" {% endfor %}
Run once per region: US, IN, DE, SG, BR.
31
Issue per-transaction mandate for Payment Execution agents
caracal cli authority mandate \ --issuer-id <regional-orchestrator-uuid> \ --subject-id <payment-execution-uuid> \ --tool-id <payment-tool-id> \ --validity-seconds 1800
Issue one mandate per payment agent at dispatch time. Validity of 30 minutes matches the transaction window.
