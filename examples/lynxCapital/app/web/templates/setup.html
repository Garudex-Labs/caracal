Caracal setup

Complete each phase in order. Sections 2–6 run inside caracal cli.

0 · Install▶
pip install caracal-core caracal-sdk
Requires Python 3.10+ and Docker.
1 · Start runtime▶
Host terminal — run in order, wait for each to succeed.
caracal bootstrap
caracal up
eval "$(caracal auth token --format env)"
Set the printed CCL_SESS_TOKEN in the Lynx environment before starting the app.
2 · Workspace▶
Enter caracal cli then paste.
workspace create lynx-capital workspace use lynx-capital
3 · Principals▶
Inside caracal cli. Note the UUID each line prints — needed in phases 5 and 6.
# Finance Control — root orchestrator; holds master authority and delegates to regions principal register --type orchestrator --name "Finance Control" --email finance-control@lynxcapital.local
# Regional Orchestrator — per-region dispatch layer; receives delegated mandates principal register --type orchestrator --name "Regional Orchestrator" --email regional-orchestrator@lynxcapital.local
# Invoice Intake — ingests and parses incoming invoice documents principal register --type worker --name "Invoice Intake" --email invoice-intake@lynxcapital.local
# Ledger Match — reconciles parsed invoices against internal ledger entries principal register --type worker --name "Ledger Match" --email ledger-match@lynxcapital.local
# Policy Check — validates each transaction against compliance and limit rules principal register --type worker --name "Policy Check" --email policy-check@lynxcapital.local
# Route Optimization — selects the optimal payment rail and external provider principal register --type worker --name "Route Optimization" --email route-optimization@lynxcapital.local
# Payment Execution — submits approved payments to external banking providers principal register --type worker --name "Payment Execution" --email payment-execution@lynxcapital.local
# Audit — records the full immutable audit trail for every transaction principal register --type worker --name "Audit" --email audit@lynxcapital.local
# Exception Investigation — investigates and resolves failed or flagged transactions principal register --type worker --name "Exception Investigation" --email exception@lynxcapital.local

Upgrade payment providers to scoped mode so tools can be registered against them.
# Mercury Bank — add scoped catalog (payment resource + actions) provider update mercury-bank --mode scoped --resource "payment=Payment account management" --action "payment:get_account_balance:GET:/v1/balance" --action "payment:submit_payment:POST:/v1/payments"
# Wise Payouts — add scoped catalog (payout resource + actions) provider update wise-payouts --mode scoped --resource "payout=Cross-border payout transfers" --action "payout:get_quote:GET:/v1/quotes" --action "payout:submit_payout:POST:/v1/payouts"
Register tools so mandates in Phase 6 can reference them. Click Fill UUIDs after Phase 3 first so <finance-control-uuid> is substituted.
# Mercury Bank payment tool — used as the root mandate tool tool register --tool-id "provider:mercury-bank:resource:payment:action:submit_payment" --provider-name mercury-bank --resource-id payment --action-id submit_payment --actor-principal-id <finance-control-uuid>
# Wise Payouts payout tool — used as the payment mandate tool tool register --tool-id "provider:wise-payouts:resource:payout:action:submit_payout" --provider-name wise-payouts --resource-id payout --action-id submit_payout --actor-principal-id <finance-control-uuid>
5 · Policies▶
Inside caracal cli. Run principal list first to get each UUID, then substitute below.
# Run this to get principal UUIDs for the commands below principal list
# Finance Control — scoped to both payment providers, delegation allowed (8-hour window) policy create --principal-id <finance-control-uuid> --max-validity-seconds 28800 --resource-pattern "provider:mercury-bank:resource:payment" --resource-pattern "provider:wise-payouts:resource:payout" --action "provider:mercury-bank:action:submit_payment" --action "provider:wise-payouts:action:submit_payout" --allow-delegation --max-delegation-network-distance 3
# Regional Orchestrator — scoped to wise-payouts, delegation allowed (6-hour window) policy create --principal-id <regional-orchestrator-uuid> --max-validity-seconds 21600 --resource-pattern "provider:wise-payouts:resource:payout" --action "provider:wise-payouts:action:submit_payout" --allow-delegation --max-delegation-network-distance 2
# Policy Check — read-only access to compliance-nexus (1-hour window) policy create --principal-id <policy-check-uuid> --max-validity-seconds 3600 --provider compliance-nexus --resource-pattern "provider:compliance-nexus:resource:*" --action "provider:compliance-nexus:action:*"
# Payment Execution — Mercury Bank payment submission (30-minute window) policy create --principal-id <payment-execution-uuid> --max-validity-seconds 1800 --provider mercury-bank --resource-pattern "provider:mercury-bank:resource:payment" --action "provider:mercury-bank:action:submit_payment"
# Payment Execution — Wise payouts submission (30-minute window) policy create --principal-id <payment-execution-uuid> --max-validity-seconds 1800 --provider wise-payouts --resource-pattern "provider:wise-payouts:resource:payout" --action "provider:wise-payouts:action:submit_payout"
