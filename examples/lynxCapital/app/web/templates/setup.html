Caracal setup

Run these commands once before starting the demo. Each block is a single terminal command.

0. Install Caracal▼
0
Install the Caracal CLI and SDK
pip install caracal-core caracal-sdk
This installs the caracal CLI and the Python SDK used by this demo app. Requires Python 3.10+ and Docker.
1. Start the Caracal runtime▼
1
Bring up the full stack
caracal up
Starts postgres, redis, vault, and the Caracal API on the default port (8080).
Permission denied on ~/.caracal?

If you see PermissionError: [Errno 13] Permission denied: '/home/…/.caracal/runtime', the directory was created by root (e.g. a prior docker run as root). Fix it with either option:

Option A — use a writable override (no sudo needed):

export CARACAL_HOME=~/.config/caracal

Then run caracal up again. Add the export to your shell profile to make it permanent.

Option B — reclaim ownership (requires sudo):

sudo chown -R $USER:$USER ~/.caracal
2
Run migrations
caracal migrate
2. Workspace▼
3
Create workspace
caracal cli workspace create --name lynx-capital --display "Lynx Capital"
4
Issue an API key for the demo app
caracal cli workspace key create --workspace lynx-capital --name demo-app
Copy the output key into CARACAL_API_KEY in your .env file.

5. Create policies▼
25
Finance Control — full provider access
caracal cli policy create --workspace lynx-capital \ --principal finance-control \ --effect allow --resource "provider:*:resource:*:action:*"
26
Policy Check agents — read-only (no payment actions)
caracal cli policy create --workspace lynx-capital \ --principal policy-check \ --effect allow --resource "provider:compliance-nexus:resource:*:action:*" policy create --workspace lynx-capital \ --principal policy-check \ --effect deny --resource "provider:mercury-bank:resource:payment:action:submit_payment"
27
Payment Execution — narrow payment scope only
caracal cli policy create --workspace lynx-capital \ --principal payment-execution \ --effect allow --resource "provider:mercury-bank:resource:payment:action:submit_payment" policy create --workspace lynx-capital \ --principal payment-execution \ --effect allow --resource "provider:wise-payouts:resource:payout:action:submit_payout"
28
All other layers — read/lookup access, no payment write
caracal cli policy create --workspace lynx-capital \ --principal regional-orchestrator \ --effect allow --resource "provider:*:resource:*:action:*"
6. Issue mandates and delegate authority▼
29
Issue Finance Control mandate
caracal cli mandate issue --workspace lynx-capital \ --principal finance-control --scope global \ --ttl 8h
30
Delegate to Regional Orchestrators (one per region)
{% for r in regions %}caracal cli mandate delegate --workspace lynx-capital \ --from finance-control --to regional-orchestrator \ --scope region:{{ r.id }} --ttl 6h {% endfor %}
Run once per region: US, IN, DE, SG, BR.
31
Delegate Payment Execution mandate template
caracal cli mandate template create --workspace lynx-capital \ --principal payment-execution \ --scope "payment:{vendor}:{amount}:{rail}:{window}" \ --ttl 30m --max-uses 1
Each ephemeral payment agent receives a one-use mandate scoped to its exact transaction tuple.
